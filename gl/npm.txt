npm init //создает package.json
Файл package.json содержит в себе информацию о вашем приложении: название, версия, зависимости и тому подобное. Любая директория, в которой есть этот файл, интерпретируется как Node.js-пакет, даже если вы не собираетесь публиковать его.
Способ использования файла package.json зависит от того, собираетесь ли вы скачивать пакет или публиковать его.

Если вы хотите использовать в своём проекте множество пакетов, то лучше указать их package.json вместо того, чтобы каждый раз скачивать их через терминал. Если какой-либо пакет имеет зависимости, то npm найдёт их через package.json загружаемого пакета и скачает их. В нашем случае у пакета canvas-chart тоже есть файл package.json с прописанными в нём зависимостями.


npm adduser //добавляет юзера, который публикует модуль
npm publish ..публикация в центрально бд модулей
npm search *key words* //поиск по названию или ключевым словам


npm install *name*//установка в папку node_modules

1) при установке, npm ищет папку node_modules в текущей директории, если не находит идет выше по дереву и ище ее там
если он найдет ее - ставит модуль в найденную папку node_modules

2) Либо npm ищет package.json (в том числе вверх по иерархии) и если находит, создает node_modules на одном уровне
с package.json

3) Если node_modules и package.json не найдены -  созадется папка node_modules текущей директории


npm install -g *name*//установка модуля в 

npm install //при наличиии package.json установит все необходимые модули
